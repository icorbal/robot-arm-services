You are analyzing images from a single camera mounted on a robot arm's end-effector. The arm has moved to two different observation poses to capture the scene from different angles, providing a wide baseline for 3D triangulation.

## Image Details
- Resolution: {width} x {height} pixels
- First image: captured from the "{pose_a}" observation pose
- Second image: captured from the "{pose_b}" observation pose
- The camera viewpoints are separated by a wide angular baseline (~58°)
- Objects are colored geometric shapes: boxes, cylinders, spheres

## Your Task
Identify ALL objects visible on the table surface. For EACH object, provide:
1. A descriptive ID (e.g., "red_box", "blue_cylinder")
2. The object's color (e.g., "red", "blue", "green", "yellow")
3. The object type (box, cylinder, sphere)
4. The pixel coordinates of the object's CENTER in the FIRST image: [u, v]
5. The pixel coordinates of the object's CENTER in the SECOND image: [u, v]

## Pixel Coordinate Rules
- Origin (0, 0) is the TOP-LEFT corner of the image
- u increases rightward (0 to {width})
- v increases downward (0 to {height})
- Estimate the CENTER of each object, not an edge or corner
- Be as precise as possible — even a few pixels matter for 3D accuracy

## CRITICAL: Multi-View Matching
- The two images are taken from VERY different angles (~58° apart)
- The same object will appear at SIGNIFICANTLY different pixel positions across images (100-300px differences in both u and v)
- Match objects across images by their COLOR and SHAPE, not by position
- Every object visible in both images must be reported with coordinates for both views

## Output Format
Return ONLY valid JSON:
```json
{
  "objects": [
    {
      "id": "red_box",
      "color": "red",
      "type": "box",
      "left_px": [320, 200],

```

## Important
- Do NOT estimate distances, sizes, or 3D positions — only pixel coordinates.
- Include every distinct object you can see in BOTH views.
- If an object is only visible in one view, skip it (both views needed for triangulation).
- If an object is partially occluded, still include it with your best estimate of its center.
